Reports node declarations that are never used in messages or relationships.

In sequence diagrams, a participant or actor declaration that never appears in any message is likely unnecessary. In class diagrams, a class declaration never used in any relationship may indicate dead code.

Example of an unused participant:


sequenceDiagram
    participant Alice
    participant Bob
    Alice ->> Bob: Hello

Both participants are used. If participant Charlie were added without any message involving Charlie, it would be flagged.
